Efficient Crowd Pathfinding

A 4-week deep dive into optimizing crowd movement logic. Solving performance bottlenecks by reducing redundant path calculations and utilizing spatial partitioning.

# Technical Architecture

Multiple World Partitioning

  • Separated physics/collision world from navigation world.
  • Collision world utilizes minimal geometry for fast intersection tests.
  • Navigation world optimized with higher detail only where agents can traverse.

Static Octree & Grouping

Octree nodes are generated based on the navmesh. This structure groups enemies sharing similar origins, allowing them to share a single calculated path.

Optimization Win

Drastically reduces A* calls by sharing paths and limiting collision checks to nearby octree nodes.

Multithreaded Pathfinding

A* algorithm with path funneling for realistic movement.

Executed on background threads to prevent main-thread blocking during heavy re-calculations.

# The Gameloop

1. Update Player
2. Pathfinding Manager
  • Prune invalid paths (empty octree nodes).
  • Queue new paths for populated nodes.
  • Deduplicate paths intersecting with existing valid ones.
3. Enemy Manager
  • Assign shared paths to octree groups.
  • Update Enemy Entities (Instanced):
    • Calculate steering (avoidance).
    • Physics updates (ground snapping).
    • Update instance buffers.
